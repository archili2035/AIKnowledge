# 00_AI知识总览

这个目录用于承载 AI 知识库的“总览层”，目标不是替代所有知识正文，而是提供一个**宏观地图入口**。

## 目录职责

- **展示层入口**：用 `index.html` 提供类似脑图 / XMind 的一览视角，帮助快速建立整体概念地图。
- **总览主树**：用 `overview.json` 存放“主树结构”，负责定义一级分类与代表节点；节点需有稳定 `id`，前台跳转统一走 HTML 详情页 `path`，底层 Markdown 来源记录在 `sourcePath`。
- **关系层**：用 `relations.json` 描述跨分支的横向关系，例如“提供方 / 竞品 / 用于 / 相关概念”；当前已进入首批样例关系阶段。
- **节点详情层**：把重要对象拆成独立 Markdown 条目，用 front matter 存元数据；前台则通过 `node.html?id=...` 这类 HTML 详情页来阅读，不再直接跳源 Markdown。


## 为什么不继续只用单文件脑图 Markdown

原始的 `../AI知识图谱.md` 很适合作为脑图转写后的全景草稿，但它同时混合了：

- 概念
- 模型
- 产品
- 公司
- 场景
- 趋势
- 个人观察

当这些内容继续增长时，单文件 Markdown 会越来越难同时承担“主数据 + 展示页 + 导航页”的职责。

## 当前建议的双层方案

### 1. 底层：可维护的数据层

- 原子化 Markdown 条目：一个对象一页
- front matter 元数据：`id / type / tags / updated / sources / related`
- 总览主树：`overview.json`
- 横向关系：`relations.json`，用于表达跨分支连接

### 2. 上层：可总览的展示层

- `index.html`：宏观总览页，负责一眼看全局
- `node.html`：正式版节点 HTML 详情页入口
- 后续专题页：模型地图 / 通用 Agent 地图 / 游戏 AI 工具链等
- 后续详情页：点开具体节点查看定义、来源、判断与相关链接

## 当前文件说明

- `index.html`：AI 知识总览页，负责宏观地图入口
- `node.html`：正式版节点 HTML 详情页入口
- `overview.json`：总览树数据；`path` 指向前台 HTML 详情页，`sourcePath` 指向底层 Markdown
- `relations.json`：横向关系层草稿（已含首批样例关系）
- `nodes/README.md`：节点详情层说明
- `nodes/TEMPLATE.md`：统一条目模板
- `nodes/`：首批样例节点目录
- `README.md`：本目录的结构说明与迁移思路


## 建议的后续迁移顺序

1. 先维护 `overview.json`，稳定一级分类与核心节点
2. 再把高频节点拆成独立 Markdown 条目
3. 最后补专题页与横向关系图

## 现阶段定位

这一版已经进入“正式前台阅读入口”阶段：总览页保留脑图感，节点详情改由 HTML 承接，Markdown 继续留在底层做可维护源文件。

这一版不是最终图谱系统，但已经能验证：

- 宏观一览是否成立
- 分类是否合理
- 后续是否值得继续拆成原子条目
